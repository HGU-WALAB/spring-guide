## 좋은 객체 지향 설계의 5가지 원칙

### **SOLID** : 로버트 마틴의 5가지 좋은 객체 지향 설계의 원칙 정리

- **SRP** : Single responsibility principle

  - 하나의 클래스는 하나의 책임만 가져야 한다.
  - 하나의 책임이라는 것은 모호하기 떄문에 **변경**을 기준으로 확인 할 수있다. **변경**이있을때 파급효과가 적다면 이 원칙을 잘 따른것이다. (ex. 변경이 있을때 하나의 클래스나 하나의 지점만 고치면 OK)

- **OCP** : Open/Closed principle

  - 확장에는 열려있으나, 변경에는 닫혀있어야 함

- **LSP** : Liskov subsitution principle
  - 프로그램의 객체는 프로그램의 정확성을 깨뜨리지 않으면서 하위 타입의 인스턴스로 바꿀 수 있어야한다.
  - 단순히 컴파일을 성공하는 것을 이야기 하는것이 아님
  - (ex. 자동차 인터페이스의 엑셀은 전진하도록, 후진하면 LSP 위반)
- **ISP** : Interface segregation principle
  - 특정 클라이언트를 위한 인터페이스 여러 개가 범용 인터페이스 하나보다 낫다.
  - (ex. 자동차 인터페이스 -> 운전 인터페이스, 정비 인터페이스로 분리)
  - 인터페이스가 명확해지고, 대체 가능성이 높아짐
- **DIP** : Dependency inversion principle
  - 구체화가 아닌 추상화에 의존해야함 -> 구현 클래스가 아닌 인터페이스에 의존해야함
  - 역할에 의존

### 객체지향과 다형성

- 객체지향의 핵심은 다형성
- 다형성만으로는 OCP, DIP를 지키는 개발을 할 수 없다.
